基础表示

登录的用户 机器的名称 家目录 可以写命令了

查看当前光标所在的目录

切换目录 cd

root用户 家目录 /root 规定
bda用户 家目录 默认【/home/bda】（可以修改）

回退家目录 三种方法（a.直接cd；b.cd ~；c.cd /root）
cd - 回退到上一次的目录
cd …/ 回退上一层目录
cd …/…/ 回退上2层目录

目录 文件夹 路径

绝对路径 /根目录 以根目录开始 Windows中：E:\opt\modules
Linux中： /opt/modules
相对路径 不以根目录为开始，以当前光标所在的目录为开始

clear 清空屏幕

ls查看当前光标所在的目录文件有哪些

ls -l（查看详细信息）

ls -l ==> 等价于ll

ls -l -a 查看当前的文件文件夹也可以看到隐藏文件文件夹（隐藏文件以.开头）
ll -h 查看文件的大小
ll -rt 按时间排序

ls -l -a ==> ls -la

如何查询命令帮助

mkdir 创建文件夹

[bda@mao01 softwares]$ mkdir a（创建一个文件夹）
[bda@mao01 a]$ mkdir -p b/c/d（创建一组文件夹路径）
[bda@mao01 d]$ mkdir 1 2 3（创建多个文件夹）

mv移动 始终只有1份 cp拷贝 原文件不动，新增一份

使用tab键可以补全文件名

（移动比复制速度要快，复制和拷贝都能做修改名称）

如何创建一个空文件 或者把一个文件设置为空

touch rz.log 创建一个空文件
echo “” > rz.log1 慎用
cat /dev/null > mao.log12 把一个文件设置为空

查看文件内容

cat 文件内容一下子全部显示
more 文件内容一页一页的往下翻，按空格键往下，回退不了，按q退出
less 文件内容 往下 往上 按上下箭头的按键 按q键退出

配置文件 内容较少时使用：cat more less
log日志 内容较多时使用：实时查看命令

tail -f xxx.log（用-f新创建的就不能匹配到）
tail -F xxx.log 就是 -f+ retry（不断尝试存不存在）
flume exec source 切记使用 -F

xxx.log log4j代码项目，配置输出的日志
规则：xxx.log 保留10份存放100M，aaa服务的日志xxx.log一直写到99m，xxx.log如果写到了100m此时就会将xxx.log进行mv变成xxx.log1，然后touch一个新的xxx.log 空文件，此时应该使用-F不断尝试，能读取到新的文件。

当ERROR 如何定位
文件内容很小 几十兆，上传给window，用editplus工具打开

上传下载 yum install -y lrzsz

文件内容很大 几百兆 2G

cat xxx.log | grep ERROR
命令输出结果 | 命令输入的 管道符
grep过滤

cat hadoop-bda-namenode-mao01.log | grep -A 10 ERROR 后10行
cat hadoop-bda-namenode-mao01.log | grep -B 10 ERROR 前10行
cat hadoop-bda-namenode-mao01.log | grep -C 30 ERROR 前后各30行 经常用 迅速定位ERROR上下文
cat hadoop-bda-namenode-mao01.log | grep -C 30 ERROR > 新建/覆盖一个新文件
cat hadoop-bda-namenode-mao01.log | grep -C 30 ERROR >> 对这个文件进行追加

注：
tail -300f messages 实时查看倒数300行文件（定位日志，重命名时有问题）
tail -300F messages 不能这样写
tail: option used in invalid context – 3
