Hadoop

狭义：
Hadoop是最重要最基础的一个部分
广义：
Hadoop生态圈：Hadoop、Hive、Sqoop、HBase…
所以基于Hadoop处理大数据存在一定弊端：
处理如下业务：join/group by如果进行面向MapReduce编程难度很大，很麻烦
RDBMS的需求

==> Hive的产生背景

MySQL/Oracle…等数据库想要进行统计操作

先创建表： 表名、列名、列类型… schema(元数据/metadata)，然后再使用SQL进行相应的业务统计分析
HDFS上的文件： 文本格式
按照某个分隔符(\t)： 第一列是什么、第二列是什么

元数据 vs 源数据

源数据： HDFS上的文件
元数据： 是描述数据（HDFS上的数据）的数据（也就是表名、列名、列类型这类数据）

Hive概述

官网：hive.apache.org
The Apache Hive ™数据仓库软件使用老进行读、写，使用SQL管理在分布式存储系统（HDFS/S3）中存储的的大型数据集，提供了一个命令行工具和JDBC驱动程序来将用户连接到Hive
Facebook开源，用于解决海量结构化的日志数据统计问题；构建在Hadoop（HDFS/MapReduce/YARN）之上的数据仓库
Hive的数据是存放在HDFS之上，Hive底层执行引擎： MapReduce/Tez/Spark，只需要通过一个参数就能够切换底层的执行引擎
Hive作业提交到YARN上运行；提供了HQL查询语言和SQL类似，但不完全相同
所谓的大数据“云化”：
是一个很大项目
适用于离线处理/批处理
HDFS上的数据：
文本、压缩、列式存储
SQL on Hadoop用到的组件有：Hive/Presto/Impala/Spark SQL…

Hive的优缺点

优点： SQL
缺点： 如果底层引擎是基于MR，性能必然不高

Hive架构：


用户接口层： 黑窗口、JDBC
Driver/驱动器
SQL解析： SQL ==> AST(antlr)
查询优化： 逻辑/物理执行计划
UDF/SerDes： 用户自定义函数和序列化
Execution： 执行
元数据： 表名、所属数据库、列（名/类型/index）、表类型、表数据所在目录
元数据包括table的db、column、location
Hive的数据是存放在HDFS之上
Hive的数据分为两部分：数据 + 元数据

Hive和RDBMS的对比
相同点：

SQL，事务，（insert/update/delete）Hive-0.14才支持，针对Hive数据仓库，写比较少的，批次加载数据到Hive然后进行统计分析

不同点：

体量/集群规模，延迟/时性

Hive部署：

查看网址：
https://cwiki.apache.org/confluence/display/Hive/GettingStarted
前置：
JDK8，Hadoop2.x，Linux，MySQL
版本：
CDH-5.16.2
wget http://archive.cloudera.com/cdh5/cdh/5/hive-1.1.0-cdh5.16.2.tar.gz

安装：

解压：
tar -zxvf hive-1.1.0-cdh5.16.2.tar.gz -C ~/app/
配置环境变量： source /etc/profice
export HIVE_HOME=/home/hadoop/app/hive-1.1.0-cdh5.16.2
export PATH=$HIVE_HOME/bin:$PATH
$HIVE_HOME下目录说明
bin： 脚本
lib： 依赖包
conf： 配置文件
$HIVE_HOME/conf/hive-site.xml

hive.cli.print.current.db这个参数设置为true
可以显示出当前的数据库
hive.cli.print.header这个参数设置为True
可以显示出表的列名
拷贝MySQL驱动包到$HIVE_HOME/lib/下

!clear; 清屏命令
exit; 退出命令
use dbname; 切换到dbname所在数据库
show tables; 查看当前数据库下的所有表
创建表： create table stu(id int, name string, age int);
查看表结构：
desc stu;（查看到的信息比较少）

desc extended stu;（查看到的信息会多一些）

desc formatted stu; （查看到的信息会格式化）

插入数据： insert into stu values(1,‘mao’,26);（但是插入数据需要跑MR非常不建议使用）
查询数据： select * from stu;

创建stu表，默认存储在HDFS的目录是

hdfs://ruozedata001:8020/user/hive/warehouse/stu
hive.metastore.warehouse.dir：/user/hive/warehouse
stu：表的名字
表的完整路径是： ${hive.metastore.warehouse.dir}/tablename

Hive的完整执行日志：

cd $HIVE_HOME
cp hive-log4j.properties.template hive-log4j.properties
里面有个参数
hive.log.dir=${java.io.tmpdir}/${user.name}
hive.log.file=hive.log
这个参数可以看到hive日志的目录（但是建议配置到其他文件夹中）
${java.io.tmpdir}/${user.name}/${hive.log.file}
/tmp/hadoop/hive.log

Hive配置属性：

全局： $HIVE_HOME/hive-site.xml中
临时/当前session（当前黑窗口）： 仅对当前session生效
查看当前的属性： set key;
修改当前的属性： set key=value;
也可以在启动时加上： hive -hiveconf hive.cli.print.current.db=true

Hive中交互式命令：

-e： 不需要进入hive命令后，就可以跟上sql语句查询
-f： 执行指定文件（内容是SQL语句）

拓展：基于Hive的离线统计/数据仓库

把SQL封装到shell脚本中去，使用hive -e "里面写sql命令"
定时调度：crontab

Hive中的数据抽象

Hive中的表(stu)必须要归属于某个数据库(default)

Database（数据库）：

包含了0到N张表
每个db对应HDFS上的一个文件夹：default==>/user/hive/warehouse

创建数据库（其中中括号表示可有可无，但是在生产中IF NOT EXISTS一定要加上）：

CREATE (DATABASE|SCHEMA) [IF NOT EXISTS] database_name
[COMMENT database_comment]
[LOCATION hdfs_path]
[WITH DBPROPERTIES (property_name=property_value, …)];

CREATE DATABASE IF NOT EXISTS mao;

自己创建的Hive数据库的存放目录规则：

${hive.metastore.warehouse.dir}/dbname.db
/user/hive/warehouse/mao.db

如果用LOCATION指定了创建数据库的路径，数据库的存放路径就在指定的目录下

CREATE DATABASE IF NOT EXISTS mao_hive LOCATION ‘/user/mao_hive’;

查看数据库的详细信息（desc databases database_name）：
创建数据库的时候加上注释和数据库的属性：

CREATE DATABASE IF NOT EXISTS mao_hive1 COMMENT ‘this is MX database’ WITH DBPROPERTIES(‘creator’=‘mx’, ‘date’=‘2019-12-18’);

修改数据库：

ALTER (DATABASE|SCHEMA) database_name SET DBPROPERTIES (property_name=property_value, …); – (Note: SCHEMA added in Hive 0.14.0)
ALTER (DATABASE|SCHEMA) database_name SET OWNER [USER|ROLE] user_or_role; – (Note: Hive 0.13.0 and later; SCHEMA added in Hive 0.14.0)
ALTER (DATABASE|SCHEMA) database_name SET LOCATION hdfs_path; – (Note: Hive 2.2.1, 2.4.0 and later)

删除数据库

DROP (DATABASE|SCHEMA) [IF EXISTS] database_name [RESTRICT|CASCADE];
DROP DATABASE IF EXISTS mao_hive;
如果数据库中有表，在删除时就会报错：
hive (mao_hive)> DROP DATABASE IF EXISTS mao_hive;
FAILED: Execution Error, return code 1 from org.apache.hadoop.hive.ql.exec.DDLTask. InvalidOperationException(message:Database mao_hive is not empty. One or more tables exist.)
此时如果想删除要使用CASCADE（但是了解即可，慎用）：一对多相当于现在的一个数据库对应多个table表

基本数据类型

HDFS上的文件来说： string
数值类型： int、bigint、float、double、decimal
字符串类型： string生产中占90%
布尔类型： boolean：true/false
日期类型： date、timestamp一般不使用，用String类型就可以满足

分隔符
分隔符 对应编码 描述
^A \001 字段之间的分隔符
\n \n 记录分隔符
^B \002 ARRAY/STRUCT（Hive中的复杂数据类型）
^C \003 MAP中的key/value（Hive中的复杂数据类型）

但是不可以和数据中的内容重合
1,pk,30
1$$$$$pk$$$$$30

create table stu(id int, name string, age int) row format delimited fields terminated by ‘,’;
insert into stu values(1,‘mao’,25);
