hadoop软件

Hdfs：
主要用于存储
Mapreduce：
做计算，使用作业，挖掘出有价值的数据主要用于数据挖掘
==> 由于开发难度高、代码量大、维护困难、计算慢，所以大家基本不会使用MR，都使用hive sql、spark、flink
Yarn：
资源（内存是VCORE） + 作业调度
海量的数据 1000台（通过yarn来进行分布式计算）

节点介绍

namenode
名称节点是老大，作用是读写请求先经过它，被称为主节点
datanode
数据节点是小弟作用是存储数据、检索数据，被称为从节点
secondarynamenode
第二名称节点是老二，恢复机制是h+1只能恢复一个小时之前的，单点的问题。
注：
大数据组件基本都是主从架构：hdfs、yarn、hbase
但是Hbase读写请求不经过老大master进程

外部界面端口是50070打开一下：http://JD:50070

Hadoop用到的命令

先查看帮助文档

具体命令详解：

hdfs dfs -mkdir -p /user/data 在hdfs中创建路径
hadoop fs -mkdir -p /user/data
hdfs dfs -ls /user/data 显示路径中的文件清单
hadoop fs -ls /user/data
hdfs dfs -put log.txt /user/data 把log.txt放进/user/data中
hadoop fs -put log.txt /user/data
hdfs dfs -cat /user/data/log.txt head
hadoop fs -cat /user/data/log.txt head
hdfs dfs -get /user/data/log.txt 从hdfs中取出文件到当地
hadoop fs -get /user/data/log.txt
hdfs dfs -rm /user/data/log.txt 删除文件，放到一个地方
hadoop fs -rm /user/data/log.txt
hdfs dfs -rm –skipTrash /user/data/log.txt 直接删除文件
hdfs dfs -rm -r -skipTrash /folder_name 删除文件夹
hdfs dfs -test /user/data/log.txt
echo $? 查看存不存在
hdfs fsck / 查看集群的详细信息
hdfs dfs -df -h 查看集群的硬盘空间情况
hdfs dfs -du / 查看每一个目录的硬盘使用情况
hdfs dfs –copyFromLocal t.txt
hdfs://localhost/user/data 从本地复制文件
hdfs dfs -copyToLocal /user/data/t.txt tt.txt 把文件复制到本地
hdfs dfsadmin -report 查看当前节点的使用情况
