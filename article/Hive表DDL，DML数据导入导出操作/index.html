where和having的区别：
  1. where只能指定行的条件，不能指定处理聚合处理后的数据，所谓聚合处理，常见的有group by和sum()，count()，avg()这些；
  2. having则用来指定这些聚合处理后的数据，简单讲，having语句一次聚合汇总之后的结果。having语句后只能出现三种要素：常数/聚合函数/group by指定的聚合键（也就是分组类别）。
    所以where语句用来指定行所对应的条件（单条），having语句用来指定组对应的条件，往往与聚合函数或group by 搭配使用。
抓取问题：（fetch）

当设置为none之后不管执行什么都会执行一个MapReduce

hive.fetch.task.conversion
•Default Value: minimal in Hive 0.10.0 through 0.13.1, more in Hive 0.14.0 and later
•Added In: Hive 0.10.0 with HIVE-2925; default changed in Hive 0.14.0 with HIVE-7397 Some select queries can be converted to a single FETCH task, minimizing latency. Currently the query should be single sourced not having any subquery and should not have any aggregations or distincts (which incur RS – ReduceSinkOperator, requiring a MapReduce task), lateral views and joins.
Supported values are none, minimal and more.

  1. none: Disable hive.fetch.task.conversion (value added in Hive 0.14.0 with HIVE-8389)
  2. minimal: SELECT *, FILTER on partition columns (WHERE and HAVING clauses), LIMIT only
  3. more: SELECT, FILTER, LIMIT only (including TABLESAMPLE, virtual columns) “more” can take any kind of expressions in the SELECT clause, including UDFs.
    (UDTFs and lateral views are not yet supported – see HIVE-5718.)
