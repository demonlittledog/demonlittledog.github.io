echo 打印

环境变量名称 $PATH

查看命令所属的路径在哪 去$PATH找 找到第一个就返回

此时并没有全局生效，作用域仅限当前session 会话（打开一个新的会话就好了）

环境变量

全局：/etc/profile 所有用户都使用
个人：~/.bash_profile
~/.bashrc 个人 不给其他人

*生效文件：*source /etc/profile（全局个人都可以）
. ~/.bashrc

安装unzip命令

（一定要做好用户组）
环境变量是指的什么
K=V 前后不能有空格 使用时 $K
修改文件的步骤：vi /etc/profile
a. 上下键 移动光标
b. 按 i键insert 进入 编辑模式
c. 开始编辑
d. 按 esc键退出 编辑模式，进入命令行模式
e. 按 shift+: 键，进入尾行模式
f. 输入 wq 保存退出

一定要记住常常用一下which看看装好没有

总结：
(1) command not found
没有部署安装包，部署了没有配置环境变量，有可能yum一装就好了
(2) 习惯
当我们以后部署一个软件，bin目录的可执行文件 比如java
习惯 当生效环境变量文件，习惯做 which java
(3) 别名
[bda@mao01 ~]$ alias
alias egrep='egrep --color=auto’
alias fgrep='fgrep --color=auto’
alias grep='grep --color=auto’
alias l.='ls -d . --color=auto’
alias ll='ls -l --color=auto’
alias ls='ls --color=auto’
alias which='alias | /usr/bin/which --tty-only --read-alias --show-dot --show-tilde’
简化操作*

.bash_profile ssh bug
.bashrc 个人环境变量文件 优先
[bda@mao01 ~]$ vi .bashrc

history

hdfs数据pb级别数据丢失通过history来查看历史记录
使用history –c来进行清空
直连比较好用
跳板机没有关系还可以使用
（跳板机直接A机器连接B机器，没有web界面的和可视化的）
堡垒机有一个log文件直接记录操作
（堡垒机也是A机器跳到B机器，有web界面，可以进行可视化，一般用来审计）

***history 看看这个账号之前做过哪些操作，有可能发现password ***

删除

文件的创建：touch mao.log
cat /dev/null > mao.log
vi命令什么都不输入，直接:wq存成空文件，也会生成新文件
文件夹的创建：mkdir

rm -f mao.log 直接删除文件
rm -rf 1 直接删除文件夹

风险：
Linux里只有文件和文件夹这两种规格；
rm -rf / 一定不能用，高危命令，一般不会这样但是在shell脚本的时候有可能发生
K=’'首先K赋值为空
rm -rf $K/
[bda@mao01 ~]$ K=’’
[bda@mao01 ~]$ echo $K/（此时就会找到根目录）
/没有回收站的（删了真的就是删了，啥都没了）*
